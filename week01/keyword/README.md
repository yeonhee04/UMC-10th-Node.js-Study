- 외래키
  **외래키 (Foreign Key)**
  : 한 테이블의 필드가 **다른 테이블의 기본키를 참조하는 키**
  → 테이블 간 관계(Relationship)를 만들어줌
  → 중복 데이터 방지
  → 데이터 무결성 유지 (없는 값 참조 불가)
  (장점) 데이터 일관성 유지, 테이블 간 연결 구조 명확
  (단점) 삽입/삭제 시 제약 발생 (속도 저하 가능)
- 기본키
  **기본키 (Primary Key)**
  : 테이블에서 각 행(row)를 **유일하게 식별하는 키**
  → 중복 불가능
  → NULL 불가능
  → 하나의 테이블에 하나만 존재
  (장점) 데이터 구분 명확, 검색 속도 향상
  (단점) 설계 잘못하면 수정 어려움
- ER 다이어그램
  **ER 다이어그램 (Entity-Relationship Diagram)**
  : 데이터베이스의 구조를 **시각적으로 표현한 다이어그램**
  (구성 요소) > Entity 개체 → 테이블 > Attribute 속성 → 컬럼 > Relationship 관계
  (장점) 구조 이해 쉬움, 설계 단계에서 오류 감소
  (단점) 복잡한 시스템에서는 다이어그램도 복잡
- 복합 키
  **복합 키 (Composite Key)**
  : 두 개 이상의 컬럼을 조합하여 만든 기본키
  → 단일 컬럼으로는 유일성 확보가 안 될 때 사용 e.g. 학번+과목코드
  (장점) 현실적인 데이터 모델링 가능
  (단점) JOIN, 조회 시 복잡, 성능 저하 가능
- 연관관계
  **연관관계 (Relationship)**
  : 테이블 간의 연결 구조
  (종류) > 1:1 일대일 > 1:N 일대다 → 가장 많음 > N:M 다대다 → 중간 테이블 필요
  (장점) 데이터 구조 확장성 증가
  (단점) 복잡한 관계는 관리 어려움
- 정규화
  [정보처리 실기*데이터베이스06강*정규화](https://youtu.be/RXQ1kZ_JHqg?si=f0OPsoOWnJXSbqca)
  **정규화 (Normalization)**
  : 데이터 중복을 제거하고 깔끔하게 만드는 과정
  (목적) 중복 제거, 이상 현상 방지 e.g. 삽입/삭제/갱신 이상
  (주요 단계) > 1NF: 컬럼 값 원자화 > 2NF: 부분 함수 종속 제거 > 3NF: 이행적 종속 제거
  (장점) 데이터 일관성 높음, 유지보수 용이
  (단점) 테이블이 많아짐, JOIN 증가에 따라 성능 저하됨
- 반 정규화
  **반 정규화 (Denormalization)**
  : 성능 향상을 위해 일부러 **중복을 허용하는 설계**
  → JOIN 줄이기 목적
  → 조회 속도 개선
  (장점) 빠른 조회 성능, 시스템 응답 속도 향상
  (단점) 데이터 중복 증가, 무결성 유지 어려움

## 🔽 IP
**IP (Internet Protocol)**
: 인터넷에 연결된 **각 장치를 식별하기 위한 고유한 주소 체계**
- 컴퓨터, 스마트폰, 서버 등 네트워크에 연결된 모든 장치는 데이터를 주고받기 위해 IP 주소 사용
- IP는 데이터를 **패킷(Packet)** 형태로 전달하며, 패킷 안에는 **출발지 IP와 목적지 IP** 정보가 포함됨
- **(장)** 전 세계 네트워크 장치 식별 가능 ← 인터넷 통신의 기본 구조
- **(단)** 신뢰성 있는 데이터 전달 보장 불가 e.g. 패킷 손실, 순서 문제

## 🔽 PORT
**Port**
: 한 컴퓨터 안에서 **어떤 프로그램이 데이터를 받을지 구분하기 위한 번호**
- 같은 서버에서 여러 서비스가 동시에 실행될 경우, IP만으로는 구분이 어렵다는 문제 해결
- e.g. HTTP : 80, HTTPS : 443, SSH : 22, SMTP : 25

| IP | PORT |
| :--- | :--- |
| 컴퓨터 위치를 찾는 역할 | 컴퓨터 안에서 실행 중인 프로그램을 찾는 역할 |
| 아파트 주소 | 동/호수 |

## 🔽 CIDR
**CIDR (Classless Inter-Domain Routing)**
: IP 주소를 효율적으로 할당하고 관리하기 위한 표기 방식
- 표기 형식 : **IP주소 / 네트워크 비트 수**
- e.g. `192.168.1.0/24`
    - `/24` 의미 : 앞 24비트 = 네트워크 주소, 나머지 8비트 = 호스트 주소
    - 즉, 하나의 네트워크에서 사용할 수 있는 주소 수는 $2^8 = 256$개
- **(장)** 필요한 만큼만 네트워크 할당 가능, IP 주소 낭비 감소, 라우팅 효율 증가

## 🔽 TCP와 UDP 차이
| | **TCP** (Transmission Control Protocol) | **UDP** (User Datagram Protocol) |
| :--- | :--- | :--- |
| **연결 방식** | 연결 지향 | 비연결 |
| **신뢰성** | 높음 → 통신 전에 3-way handshake 과정으로 연결을 확인함 (데이터 전송 보장) | 낮음 |
| **속도** | 느림 | 빠름 |
| **데이터 속도** | 보장 | 보장 X |
| **사용 예** | 웹, 이메일 | 스트리밍, 게임 |

## 🔽 Web Server와 WAS의 차이
( Client → Web Server → WAS → DB )

| | **Web Server** | **WAS** (Web Application Server) |
| :--- | :--- | :--- |
| **역할** | 정적 콘텐츠 처리<br>e.g. HTML, CSS, 이미지 | 동적 콘텐츠 처리<br>e.g. 로그인 처리, API 실행, DB 조회 |
| **기능** | HTTP 프로토콜을 이용해 클라이언트에게 웹 페이지 제공 | 웹 애플리케이션을 실행하여 비즈니스 로직 처리, 데이터베이스와 연동 |
| **대표 서버** | Apache | Tomcat |

## 🔽 HTTP와 HTTPS의 차이
| | **HTTP** | **HTTPS** |
| :--- | :--- | :--- |
| **정의** | 웹에서 클라이언트와 서버가 데이터를 주고받기 위한 통신 프로토콜 | HTTP에 SSL/TLS 암호화를 추가한 프로토콜 |
| **기본 포트** | 80 | 443 |
| **특징** | • 데이터 암호화 없음<br>• 속도 빠름<br>• 보안 취약 | • 데이터 암호화<br>• 보안 강화<br>• 인증서 필요 |

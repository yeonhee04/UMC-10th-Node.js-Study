**1. 너디너리 홈페이지 접속 과정**

1. 사용자의 브라우저(클라이언트)는 접속 요청을 보내기 위해 **서버의 IP 주소와 PORT 번호**를 확인한다. 이때 `neordinary.co.kr`과 같은 도메인은 실제로는 IP 주소로 변환되어 사용된다.

2. 이후 클라이언트는 서버와의 안정적인 통신을 위해 **TCP의 3-way handshake 과정**을 통해 연결이 가능한 상태인지 먼저 확인한다. 이 과정을 통해 서버가 정상적으로 요청을 받을 수 있는 상태인지 검증한다.

3. 연결이 완료되면, 클라이언트는 요청 데이터(예: 웹 페이지 요청)를 **패킷(Packet)** 형태로 만들어 전송한다. 이 패킷에는 **출발지와 목적지의 IP 주소, PORT 번호** 등의 정보가 포함된다.

4. 패킷은 사용자의 컴퓨터에서 **Access Network**(와이파이, 인터넷)를 통해 인터넷으로 전달되고,
   이후 **Network Core의 라우터들**을 거치면서 목적지 서버까지 이동한다.
   이때 라우터는 패킷에 포함된 **IP 주소를 기준으로 최적의 경로를 선택**하여 전달한다.

5. 패킷이 서버에 도착하면, 서버는 먼저 **IP 주소를 확인하여 자신에게 온 요청인지 판단**하고,
   이후 **PORT 번호를 확인하여 해당 요청을 처리할 프로그램(웹 서버)**으로 전달한다.
   예를 들어 HTTPS 통신은 보통 **443번 포트**를 사용한다.

6. 서버는 요청을 처리한 뒤, 결과(HTML, CSS 등)를 다시 패킷에 담아 클라이언트에게 전송한다. 이 응답 역시 동일하게 인터넷을 통해 전달되며, 클라이언트는 이를 받아 브라우저에 웹 페이지를 출력하게 된다.
